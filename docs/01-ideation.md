# 01. チームでアイデアを作る

3名程度のチームで、LINEミニアプリとして作りたい小さなWebアプリを考えます。

いきなり仕様書を書く必要はありません。まずは付箋と絵で発散・収束し、最後にCopilotへ渡す短いメモへ変換します。

## 目標

GitHub Copilotに渡せる **アイデア引き継ぎパッケージ** を作ります。

```text
work/
  idea.md
  ui-sketch.png
  scene-1.jpg
  scene-2.jpg
```

`ui-sketch.png` が画面のスケッチです。`scene-1.jpg` などのシーン画像は、アイデアが使われる場面を伝える追加コンテキストです。どちらも名前が違っても構いません。

## 進め方

### 1. 付箋にアイデアを出す

数分間、付箋にアイデアを書きます。

- 1枚の付箋に1アイデアを書く
- 実現性を気にしすぎず、まず数を出す
- LINE上で使うと少し便利、楽しい、話題になるものを歓迎する

### 2. 投票して1つに絞る

チームで良いと思うアイデアに投票し、今回作るアイデアを1つ選びます。

選ぶときは、面白さだけでなく、ハンズオン中に小さく作れそうかも見ます。

### 3. 役に立っているシーンを描く

選んだアイデアが、どんな場面で使われているかを1コマ絵で描きます。

A3用紙を四つ折りにして4つのシーンを描くと進めやすいですが、4つ全部出なくても構いません。逆に、良いシーンが多ければ4つ以上描いても大丈夫です。

ここで描くシーンは、アプリの画面設計ではありません。アイデアの深さを出すために、利用者、状況、うれしい結果が伝わる場面を描きます。

- シーン同士は独立していてよい
- 使う人と状況が分かるようにする
- そのアイデアが役に立つ理由が分かるようにする
- スマートフォン画面だけでなく、使っている場面全体を描いてよい

描いたシーンは写真に撮って `work/` に置き、`work/idea.md` にファイル名を書きます。

### 4. UIスケッチを描く

アプリとして作る画面のラフなUIスケッチを描きます。

ここで描くUIスケッチは、Copilotに画面構成を伝える主な画像です。きれいに描く必要はありません。1〜3画面に絞り、ボタン、入力欄、結果表示など、主要な要素が分かれば十分です。

描いたUIスケッチは写真に撮って `work/` に置き、`work/idea.md` の「UIスケッチ」にファイル名を書きます。

### 5. 任意でストーリーを描く

時間があれば、独立したシーンの中からよさそうなものを選び、利用前後の一連の流れを描きます。

これは任意です。ストーリーまで描けなくても、UIスケッチ画像と `work/idea.md` があれば次に進めます。

### 6. `idea-capture-coach` で `work/idea.md` に落とす

最後に、選んだアイデア、UIスケッチ、役に立っているシーン、画像ファイル名を `work/idea.md` に短く書きます。

手で埋めても構いませんが、今回のハンズオンでは `idea-capture-coach` Agentに聞き取り役を任せると進めやすいです。VS CodeのChatでAgentを `idea-capture-coach` に切り替え、次のように依頼します。

```text
選んだアイデア、UIスケッチ、シーンスケッチ、必要ならストーリーをもとに work/idea.md を埋めてください。
足りない情報があれば、必要最小限だけ質問してください。
```

全部をきれいに埋める必要はありません。空欄があっても、次の章でCopilotが安全な仮定を置いて実装計画に変換します。

## アイデアの絞り方

70分で完成に近づけるため、以下を意識します。

- 1〜3画面で終わる
- 1つの楽しい体験、または1つの課題解決に絞る
- ログイン、決済、データベース、外部APIを必須にしない
- LINEらしさはMockプロフィール、結果カード、シェア風UIなどで表現する

## おすすめテーマ

- 診断・おすすめアプリ
- イベントチェックインカード
- スタンプカード
- クイズ・投票
- プロフィールを使ったパーソナライズ体験

## Copilotへの相談例

付箋のアイデア出しに詰まったら、Copilot Chatで次のように相談できます。

```text
3名チームでLINEミニアプリのアイデアを考えています。
70分でAIが実装できる、1〜3画面の小さなWebアプリ案を5つ出してください。
イベント参加者が使って楽しいものにしたいです。
```
