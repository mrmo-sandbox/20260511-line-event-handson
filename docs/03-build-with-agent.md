# 03. Copilotでゼロから実装する

ここでは、02で作った実装計画をもとに、`/web-coder` で `app/` 配下へWebアプリを生成します。

自分で最初からコードを書くのではなく、Copilotに実装を任せ、必要な確認だけ人間が行います。

## この章でやること

- `/web-coder` に実装計画を渡す
- `app/` 配下にVite + React + TypeScriptのアプリを作る
- まず通常のWebアプリとして主要フローを動かす
- LIFF Mockを薄い境界に分けてLINEらしさを足す
- `npm run build` が通るところまで整える
- 次の章でブラウザ確認できる状態にする

## 実装を依頼する

02で作った計画をChat上で見える状態にして、Awesome由来の実装Skillを使います。

```text
/web-coder

work/idea.md、UIスケッチ、直前の実装計画に沿って、app/ 配下に必要最低限のWebアプリを実装してください。
Vite + React + TypeScript、モバイルファーストでお願いします。
まず通常のWebアプリとして主要フローを動かし、その後にLIFF MockプロフィールでLINEらしさを足してください。
LIFF MockはUIに直書きせず、将来LINEミニアプリチャネルの開発用LIFF IDで初期化するLIFF SDKへ差し替えやすい薄い境界にしてください。
```

Copilotがファイル作成やコマンド実行の確認を求めたら、内容を見て進めます。`app/` 配下に作る提案であれば、このハンズオンの前提に合っています。

## 実装中に見るポイント

生成が進んでいる間は、細かいコードをすべて読む必要はありません。まず次の点だけ確認します。

- `app/` 配下に作っているか
- `package.json` が作られているか
- `npm run dev` と `npm run build` が用意されているか
- 画面数と機能が、02の計画より大きくなりすぎていないか
- LIFF MockがUIに直書きされず、プロフィール取得などの境界に分かれているか
- LIFF IDやAPIキーなどの秘密情報を直接書いていないか

## 生成後に実行するコマンド

アプリが生成されたら、ターミナルで確認します。

```bash
cd app
npm install
npm run build
```

`npm run build` が通ったら、開発サーバーを起動します。

```bash
npm run dev
```

Codespacesでは、Portsパネルから5173番を開くと確認しやすいです。ローカルVS Codeでは、ターミナルに表示されたURLを開きます。

## エラーが出たとき

ビルドや起動でエラーが出たら、エラー全文をCopilotに渡します。

```text
このエラーを読んで原因を特定し、最小の修正で npm run build が通るようにしてください。
```

実装が大きくなりすぎた場合は、範囲を戻します。

```text
今の実装は大きすぎます。
02の計画に戻して、主要フローだけ残してください。
```

## 次の章へ進む目安

次の状態になったら、04のブラウザ確認へ進みます。

- `app/` が作成されている
- `npm install` が完了している
- `npm run build` が通っている
- `npm run dev` で開発サーバーが起動している
