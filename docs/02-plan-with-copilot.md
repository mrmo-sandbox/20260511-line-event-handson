# 02. Copilotで実装計画に変換する

ここでは、`work/idea.md` とUIスケッチをもとに、GitHub Copilotに **実装しやすい小さな計画** を作ってもらいます。

まだコードは書きません。次の章で `/web-coder` に渡せる状態まで、作るものを絞るのが目的です。

## この章でやること

- アイデアを1〜3画面のWebアプリに絞る
- LIFF Mockで表現するLINEらしさを決める
- 必須機能と発展課題を分ける
- `/web-coder` に渡す実装指示を作る

## 事前に用意するもの

1. `work/idea.md` をチームのアイデアで埋めます。
2. UIスケッチ画像を `work/` に置きます。
3. `work/idea.md` の「UIスケッチ」に、画像ファイル名を書きます。

画像ファイル名は `ui-sketch.png` でなくても構いません。`wireframe.jpg` や `screen.png` など、実際に置いた名前を書いてください。

## `/plan-from-idea` を実行する

VS CodeでCopilot Chatを開き、今回のハンズオン用に用意した独自Promptを実行します。

```text
/plan-from-idea
```

UIスケッチ画像をChatに添付できる場合は、添付してから実行します。添付できない場合は、`work/idea.md` の「UIスケッチ」に画面の説明を短く追記してから実行してください。

このPromptは、Awesome由来Skillsにない「ideaとUIスケッチをハンズオン向けの小さな実装計画へ変換する」ためのカスタマイズ例です。

## Copilotの出力を見るポイント

出力された計画を、そのまま採用する前に確認します。

- 作るアプリを一言で説明できるか
- 画面数が1〜3画面に収まっているか
- ユーザーの流れが最後までつながっているか
- `app/` 配下に作る前提になっているか
- Vite + React + TypeScriptで作れる内容になっているか
- LIFF MockでLINEらしさを表現できているか
- バックエンド、DB、有料API、実LIFF接続が必須になっていないか
- `npm install`, `npm run dev`, `npm run build` が想定されているか
- ブラウザ確認まで含まれているか

## 計画が大きすぎるとき

計画が大きい場合は、Copilotに小さくしてもらいます。

```text
今回はハンズオン中にデモできる最小構成にしたいです。
主要フローだけ残して、画面数と機能を減らしてください。
```

外部APIや実LIFF接続が前提になっている場合は、Mockに寄せます。

```text
外部APIと実LIFF接続は今回の必須から外してください。
LIFF Mockプロフィールと画面内のサンプルデータだけで動く計画にしてください。
```

## 次の章へ渡すもの

最後に、Copilotの出力のうち **`/web-coder` に渡す実装指示** を確認します。

次の章では、その実装指示を見える状態にしたまま `/web-coder` を使います。計画が長い場合は、最後にこう依頼して短くまとめます。

```text
次に /web-coder に渡すため、実装指示だけを短くまとめてください。
```
