# 02. Copilotで実装計画に変換する

ここでは、`work/idea.md` とUIスケッチをもとに、必要に応じてシーン画像も参照しながら、GitHub Copilotに **実装しやすい小さな計画** を作ってもらいます。

まだコードは書きません。次の章で `/web-coder` に渡せる状態まで、作るものを絞るのが目的です。

## この章でやること

- アイデアとUIスケッチを1〜3画面のWebアプリに絞る
- Webアプリとして動く主要フローを先に決める
- その後にMockで足すLINEらしさを決める
- 将来、LINEミニアプリチャネルを作り、開発用LIFF IDとエンドポイントURLを設定して差し替えやすい境界を決める
- 必須機能と発展課題を分ける
- `/web-coder` に渡す実装指示を作る

## 事前に用意するもの

1. `work/idea.md` をチームのアイデアで埋めます。迷う場合は、先に `idea-capture-coach` Agentで聞き取りながら整理します。
2. UIスケッチ画像を `work/` に置きます。
3. `work/idea.md` の「UIスケッチ」に、UI画像のファイル名を書きます。
4. シーン画像がある場合は、`work/idea.md` の「追加コンテキスト: 役に立っているシーン」に、シーンの内容と画像ファイル名を書きます。

画像ファイル名は `ui-sketch.png` でなくても構いません。`wireframe.jpg` や `screen.png` など、実際に置いた名前を書いてください。シーン画像の数も固定ではありません。

## `/plan-from-idea` を実行する

VS CodeでCopilot Chatを開き、今回のハンズオン用に用意した独自Promptを実行します。

```text
/plan-from-idea
```

UIスケッチ画像やシーン画像をChatに添付できる場合は、添付してから実行します。添付できない場合は、`work/idea.md` の「UIスケッチ」や「追加コンテキスト: 役に立っているシーン」に内容を短く追記してから実行してください。

このPromptは、Awesome由来Skillsにない「`work/idea.md` とUIスケッチ、必要に応じたシーン画像をハンズオン向けの小さな実装計画へ変換する」ためのカスタマイズ例です。`idea-capture-coach` が聞き取った内容を、ここで実装可能な計画へ絞ります。

## Copilotの出力を見るポイント

出力された計画を、そのまま採用する前に確認します。

- 作るアプリを一言で説明できるか
- 画面数が1〜3画面に収まっているか
- UIスケッチの画面構成が反映されているか
- シーン画像がある場合、そのすべてを実装対象にせず、主要フローに効く場面を選んでいるか
- ユーザーの流れが最後までつながっているか
- `app/` 配下に作る前提になっているか
- Vite + React + TypeScriptで作れる内容になっているか
- まず通常のWebアプリとして動く計画になっているか
- LIFF MockがUIに直書きではなく、プロフィール取得などの薄い境界に分かれているか
- Mockプロフィール、結果カード、共有文言でLINEらしさを表現できているか
- バックエンド、DB、有料API、LINEミニアプリチャネル設定が必須になっていないか
- `npm install`, `npm run dev`, `npm run build` が想定されているか
- ブラウザ確認まで含まれているか

## 計画が大きすぎるとき

計画が大きい場合は、Copilotに小さくしてもらいます。

```text
今回はハンズオン中にデモできる最小構成にしたいです。
主要フローだけ残して、画面数と機能を減らしてください。
```

外部APIやLINEミニアプリチャネル設定が前提になっている場合は、Mockに寄せます。ただし、将来LINEミニアプリチャネルを作り、開発用LIFF IDとエンドポイントURLを設定して差し替えやすい形にします。

```text
外部APIとLINEミニアプリチャネル設定は今回の必須から外してください。
まず通常のWebアプリとして動かし、その後にLIFF Mockプロフィールと画面内のサンプルデータでLINEらしさを足す計画にしてください。
LIFF MockはUIへ直書きせず、将来LINEミニアプリチャネルの開発用LIFF IDで初期化するLIFF SDKに差し替えられる薄い境界にしてください。
```

## 次の章へ渡すもの

最後に、Copilotの出力のうち **`/web-coder` に渡す実装指示** を確認します。

次の章では、その実装指示を見える状態にしたまま `/web-coder` を使います。計画が長い場合は、最後にこう依頼して短くまとめます。

```text
次に /web-coder に渡すため、実装指示だけを短くまとめてください。
```
