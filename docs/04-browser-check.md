# 04. ブラウザで確認して改善する

生成されたアプリをブラウザで見て、GitHub Copilotに改善してもらいます。

## 目標

ただコードを生成して終わりにせず、Agentに画面を確認させて改善ループを回します。

## 手順

1. `app/` で開発サーバーを起動します。

```bash
cd app
npm run dev
```

2. Codespacesの場合は、Portsパネルの5173番を開き、VS Code内ブラウザまたはポートプレビューで確認します。
3. Playwright MCPが使える場合は、Copilotにも画面確認を依頼します。

```text
/improve-with-browser
```

4. スマートフォン幅で、主要フローを最後まで確認します。
5. 問題が見つかったら、Copilotに修正させます。

## 見るポイント

- 最初の画面で何をすればよいか分かるか
- スマートフォン幅で文字やボタンが見やすいか
- ボタンを押したときに期待通り進むか
- 結果画面や完了画面がデモしやすいか
- LINEミニアプリらしさが伝わるか

## 改善指示の例

```text
スマートフォン幅だと結果カードが窮屈なので、余白と文字サイズを調整してください。
```

```text
初回表示で何をするアプリか分かりにくいです。ヘッダーと説明文を改善してください。
```

```text
LINEプロフィールMockをもっと自然に画面へ反映してください。
```
